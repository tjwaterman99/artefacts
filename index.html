artefacts¶

A python deserialization library for dbt artifacts.

Introduction¶

The artefacts API aims to provide a set of simple, well documented Python objects for building functionality into your dbt project by using dbt’s artifacts 1 2. The project’s goal is to make it easier to build features including linters, CI/CD runners, notifications, and others.

Here’s an example that shows how to use the API to check that all models in a project have at least 1 test.

>>> import artefacts.api
>>> models = artefacts.api.models()
>>> for model in models:
...   assert len(model.tests) > 0

There are other methods in the artefacts.api you can use to interact with your dbt project. Please see the API documentation for more details, as well as the Reference for detailed information about the various objects returned by the API.

To get started, please see the Installation instructions. We also provide a Tutorial that demonstrates how to use artefacts to build a “conventions linter” for your dbt project.

Alternatives¶

There are several alternatives to using a library like artefacts for working with your dbt project. We discuss some of the trade-offs associated with those alternatives below.

Using dbt-core¶

The dbt-core library is likely the better long term solution to using artifacts, and is already being used by projects like fal.

But currently dbt does not support a Python API for interacting with your dbt project 3. The lack of first-class support means there is no documentation for using the various Python objects provided by dbt-core, so you’ll need to read and understand the complex Python source code to accomplish simple tasks like iterating over the models in your project.

While this is possible, we felt that there was a benefit to having an easier way to programmatically interact with your dbt project.

Eventually we expect that features like the work-in-progress dbt.lib will eventually supplant the benefits of a 3rd party library like artefacts. But until features like that are released and supported, we feel a library like artefacts can be useful for analytics engineers that want to start building new functionality into their dbt projects right now.

Using dbt’s jinja context¶

Another way to interact programmatically with your dbt project is by using the Jinja context available to a dbt model. This is the approach taken by some dbt packages including dbt_meta_testing.

The Jinja context contains python objects like graph, which can be used for accomplishing tasks like identifying the models in your project that are missing documentation.

But using the Jinja context is often more complex than it might seem. Compare the following two code examples, both of which print the models in a specific package that have an incremental materialization.

{% if execute %}
{% for node in graph.nodes.values()
    | selectattr("resource_type", "equalto", "model")
    | selectattr("package_name", "equalto", "snowplow") %}

    {% do log(node.unique_id ~ ", materialized: " ~ node.config.materialized, info=true) %}

{% endfor %}
{% endif %}

import artefacts.api
for model in artefacts.api.models();
    if (
        model.package_name == 'snowplow' and
        model.config.materialized == 'table'
    ):
        print(f"{model.unique_id} ~ materialized: {model.config.materialized}")

While an “embedded” language like Jinja is fantastic for features like writing macros, it is challenging to for moderately complex projects. We feel that using Python will make it easier and faster to program new functionality into your project.

Using json.load¶

The last way we’ll discuss for programmatically interacting with your dbt project is by using the raw artifacts directly.

You can load and parse the raw artifacts with Python’s built in json library.

import json

with open('target/manifest.json') as fh:
    manifest = json.load(fh)

The manifest above is a simple Python dictionary, and that approach is probably the simplest possible for various scripts.

A deserialization library like artefacts essentially extends this approach, by converting that simple Python dictionary manifest into a more full-featured Python object.

The more full-featured Python object can then make it trivial to perform tasks like traversing the graph of dbt nodes. For example, you can easily pull data from the catalog into an object from the manifest.

import artefacts.api

model = artefacts.api.models()[0]

for column_name, column_details in model.catalog.columns.items():
    print(column_name, column_details.node_type)
